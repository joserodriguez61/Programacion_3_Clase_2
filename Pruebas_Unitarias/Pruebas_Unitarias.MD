# Pruebas Unitarias

Las pruebas unitarias se centran en probar pequeñas partes de código como funciones, métodos o clases, de forma aislada.
El objetivo de las pruebas unitarias es aislar cada unidad y probarla de manera independiente para garantizar que cumple con los requisitos y produce los resultados esperados.

Existen tres pasos:

### Acomodar

En este paso se establecen las precondiciones para la ejecucción de las pruebas unitarias. Algunos ejemplos pueden ser: la inicialización de variables, objetos o condiciones específicas que se deseen probar. 

### Accionar

En este paso se ejecuta el código que se desea probar utilizando las condiciones definidas en el paso de Acomodar.

### Asegurar

En este paso se verifica el resultado de la ejecucción del código del paso de Accionar. En caso de cumplir con el resultado esperado se acepta la prueba unitaria, caso contrario se procede a corregir el código hasta que se logre cumplir con el objetivo de la prueba unitaria.
